# Hyper-V Checkpoint erstellen

Teilnehmer VMs:
192.168.100.2	pfsense <--- OpenVPN Server
192.168.100.14	kali  <--- OpenVPN Client

Zuerst, fügen wir einen weiteren "Privaten" virtuellen Switch hinzu; benennen diesen "Kali_pfsense_direct":

VMs herunterfahren:

```powershell
New-VMSwitch -Name "Kali_pfsense_direct" -SwitchType Private
```
```powershell
Add-NetworkAdapter -VMName "pfsense" -SwitchName "Kali_pfsense_direct" -Name "Network Adapter 2"
```
```powershell
Add-NetworkAdapter -VMName "kali" -SwitchName "Kali_pfsense_direct" -Name "Network Adapter 2"
```

Wir wehlen die kali-VM um das Szenario zu test.
